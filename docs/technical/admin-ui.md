# Admin UI — управление AI-агентами в Mini App

## Описание
UI-часть админ-панели в Mini App. Поверх API `/api/admin/agents` (см. [ai-agents.md](./ai-agents.md)). Позволяет админу из Telegram Mini App создавать, редактировать, удалять AI-агентов и быстро включать/выключать их без захода в форму.

## Расположение файлов
Только `frontend/index.html`:
- объект `AdminAgents` (JS) — загрузка/рендер списка, форма, маскировка ключа, тумблер `active`;
- DOM-блок «🤖 AI-агенты» внутри `<section id="screen-admin">` — кнопка «+ Добавить» и контейнер `#admin-agents-list`;
- модальный экран `<div id="agent-form-screen" class="onboarding">` (форма создания/редактирования) — внутри `<div class="app">`, рядом с навигацией.

## Как работает
- Доступ к экрану — кнопка «Админ-панель» в `screen-profile`, видна если `Auth.isAdmin() === true`.
- При входе в `screen-admin` вызывается `AdminAgents.load()` — `GET /api/admin/agents`, рендерит карточки.
- Создание/редактирование — фуллскрин-форма `agent-form-screen` (тот же паттерн что `onboarding-screen`).
- Тумблер активности (кнопка «Вкл»/«Выкл» на карточке) — `PUT` с `{active: bool}`, без открытия формы.
- Удаление — кнопка в форме редактирования, с `confirm`.

## Безопасность
- Все запросы идут через `Api.request` с `X-Telegram-Init-Data`. Если пользователь не админ — бэк отдаёт 403.
- API-ключи никогда не отображаются в UI открытым текстом — только маска `AIza***azBc` в подсказке к полю.
- При редактировании поле API Key всегда пустое; если оставить пустым — ключ не меняется. Это страховка от случайной перезаписи и от утечки ключа на скриншоте.

## Что не делает
- Нет валидации формы на стороне фронта кроме базовой (params как JSON, обязательный apiKey при создании). Остальную валидацию делает бэк, ошибки отображаются как есть.
- Нет проверки/тестирования агента (раньше была кнопка «Проверить» для Gemini-ключа — её удалили вместе с легаси).
- Нет drag-and-drop для priority, нет batch-операций, нет фильтров/поиска.
